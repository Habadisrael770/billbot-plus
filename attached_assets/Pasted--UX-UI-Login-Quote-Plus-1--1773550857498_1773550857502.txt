# אפיון UX/UI – עמוד התחברות (Login) | Quote Plus

---

## 1. סקירת המצב הקיים

העמוד הנוכחי בנוי כ-Single Card Layout ממורכז על רקע כהה (`#11172e`), עם כרטיס (`#1f263f`) שמכיל את כל רכיבי הטופס. הכיוון הוא RTL. העמוד רספונסיבי אך משתמש באותו Layout בדיוק גם בדסקטופ וגם במובייל — הכרטיס תמיד ממורכז עם `max-width: 560px` (md).

---

## 2. מבנה היררכי — ארכיטקטורת העמוד

### שכבה עליונה (Page Wrapper)

**Container:** `min-h-screen`, `flex`, `items-center`, `justify-center`, `p-4`
**רקע:** `--background: #11172e` (כחול כהה מאוד)
**כיוון:** RTL

### שכבת הכרטיס (Card)

**מאפיינים:** `rounded-2xl`, `border` (`rgba(255,255,255,0.15)`), `shadow-sm`, `p-8` (40px), `w-full`, `max-w-md` (560px)
**רקע:** `--card: #1f263f`
**צבע טקסט:** `--card-foreground: #f8fafc`

## 3. רכיבי העמוד — מלמעלה למטה

### 3.1 לוגו

- **רכיב:** `<img>` — Quote Plus Logo
- **גודל:** `h-12`, `w-auto`
- **מיקום:** ממורכז (inline-flex, items-center, shrink-0)
- **מרווח:** חלק מאזור ה-header עם `mb-8`

### 3.2 כותרת ראשית

- **טקסט:** "התחברות למערכת"
- **רכיב:** `<h1>`
- **סגנון:** `text-3xl`, `font-bold`, `text-foreground`, `mb-2`
- **יישור:** `text-center`

### 3.3 כותרת משנה

- **טקסט:** "התחבר לחשבון שלך"
- **רכיב:** `<p>`
- **סגנון:** `text-muted-foreground` (צבע מעומעם)
- **יישור:** `text-center`

### 3.4 טופס (Form)

**Layout:** `space-y-6` (רווח אנכי 24px בין אלמנטים)

| מאפיין | ערך |
|---|---|

| Type | `email` |
| Placeholder | `example@email.com` |
| ID | `email` |
| גובה | 60px (`h-12`) |
| רוחב | `w-full` |
| פונט | `text-base`, `font-medium` |
| אייקון | אייקון מעטפה (mail) בצד שמאל |
| Border Radius | 0px (ככל הנראה מוגדר דרך parent wrapper) |

#### שדה סיסמה

| מאפיין | ערך |
|---|---|
| Label | "סיסמה" |
| Type | `password` |

| אייקון | אייקון מנעול + כפתור toggle (הצג/הסתר סיסמה) |
| לינק עזר | "שכחת סיסמה?" — מוצג מעל השדה, בצד שמאל (כיוון RTL) |

**לינק "שכחת סיסמה?":**
- סגנון: `text-sm`, `text-brand-primary`, `font-medium`
- Hover: `text-brand-primary/80` (שקיפות 80%)

### Layout

```
│  │  [═══════════ ✉]  │  │
│  │                   │  │
│  │  [סיסמה] [שכחת?] │  │
│  │  [═══════════ 🔒] │  │
│  │                   │  │
│  │  [  התחברות  ]    │  │

│  └───────────────────┘  │
│                         │
└─────────────────────────┘
```

### מאפיינים ספציפיים למובייל

- **Padding חיצוני:** `p-4` (16px) — מבטיח שהכרטיס לא נוגע בקצוות
- **Padding פנימי:** `p-8` (40px) — אפשר לשקול הקטנה ל-`p-6` (24px) במובייל
- **כרטיס:** `w-full` — תופס את כל הרוחב הזמין (מינוס padding)
- **שדות קלט:** גובה 60px — מצוין למגע (עומד ביעד מינימום 44px)
- **כפתורים:** `w-full` — מלא רוחב
- **גודל פונט:** `text-base` (16px) — מונע zoom אוטומטי ב-iOS

## 7. States & Interactions

### שדות קלט

| State | סגנון |
|---|---|
| Default | רקע `--input` (#253056), border מעומעם |
| Focus | Ring כחול-סגול (`--ring: #5a75dc`), border מודגש |
| Error | Border אדום (`--destructive`), הודעת שגיאה מתחת |
| Filled | טקסט לבן (`--card-foreground`) |

### כפתור התחברות

| State | סגנון |
|---|---|
| Default | רקע brand-primary, טקסט לבן |
| Hover | רקע מעט כהה יותר / שקיפות |
| Loading | Spinner / אנימציה, disabled |
| Disabled | שקיפות 50%, cursor not-allowed |

### כפתור Google

| State | סגנון |
|---|---|
| Default | רקע שקוף, border לבן מעומעם |
| Hover | רקע מעומעם קל |

## 8. המלצות שיפור UX

**מה עובד טוב כרגע:**
שימוש ב-Dark Theme עקבי, היררכיה ויזואלית ברורה, שדות קלט בגודל מתאים למגע, תמיכת RTL מלאה, ו-Social Login (Google) מוצע כאלטרנטיבה.

**מה אפשר לשפר:**

הכרטיס בדסקטופ נראה זהה למובייל — אפשר לשקול Split Layout עם אילוסטרציה או branding בצד אחד והטופס בצד השני, כפי שנהוג ב-SaaS מודרניים. זה ינצל טוב יותר את שטח המסך הגדול.

ה-border-radius של שדות הקלט כרגע הוא 0px, מה שלא תואם את ה-rounded-2xl של הכרטיס. מומלץ להוסיף `border-radius` עקבי (למשל 8px) לשדות.

חסרה אינדיקציית חוזק סיסמה — כדאי להוסיף בעמוד ההרשמה (לא בהתחברות).

כדאי להוסיף "Remember me" checkbox כדי לאפשר למשתמשים להישאר מחוברים.

הלינק "שכחת סיסמה?" מופיע מעל שדה הסיסמה — מבחינת UX מקובל יותר למקם אותו מתחת לשדה או מתחת לכפתור ה-Submit, כך שהמשתמש יראה אותו רק אחרי שנכשל.

כדאי לוודא שיש אנימציית loading על כפתור ה-Submit למניעת לחיצות כפולות, וכן validation בזמן אמת עם הודעות שגיאה ברורות בעברית.
